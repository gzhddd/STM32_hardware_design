# STM32_Hardware_Design
## 硬件电路总体架构

本项目基于一款经典的 Cortex-M3 最小系统核心板设计。注意：虽然常被称为 STM32 最小系统板，但根据原理图，本项目实际采用的主控芯片为 **GD32F103C8T6**（与 STM32F103C8T6 引脚与基础功能高度兼容）。整个硬件系统主要由以下几个核心功能模块组成：

### 1. 主控单元 (MCU)
* **核心芯片**：采用 GD32F103C8T6 微控制器。
* **时钟系统**：外部配置了双晶振，包含用于系统主时钟的高频晶振（OSC_IN / OSC_OUT） 和用于 RTC 实时时钟的 32.768KHz 低频晶振（OSC32_IN / OSC32_OUT）。
* **程序下载与调试**：引出了标准串行调试接口（SWD），包含 SWDIO 和 SWCLK 引脚，支持使用 ST-Link 或 DAP-Link 进行代码下载与在线仿真。

### 2. 电源管理 (Power Supply)
* **电压转换**：采用 AMS1117-3.3 线性稳压 LDO 芯片。该芯片将外部输入的 5V 电压（VIN_5V） 降压并转换为系统所需的稳定 3.3V 直流电压（VCC_3V3）。
* **电源指示**：在 3.3V 输出端配置了全局电源指示灯 LED1。
* **滤波设计**：电源的输入和输出端并联了多个滤波电容（包含 22uF、10uF 和 100nF），有效滤除高低频噪声，保障主控芯片稳定运行。

### 3. USB 接口 (USB Interface)
* **接口类型**：板载标准的 Micro USB 接口（USB1）。
* **主要功能**：该接口不仅提供整个系统的 5V 供电输入（VIN_5V），还将 USB 数据引脚（D+ 和 D-）引出，方便进行 USB 通信功能的开发。

### 4. 复位与唤醒电路 (Reset & Wakeup)
* **硬件复位**：采用按键复位与 RC 延时复位结合的标准电路，复位按键连接至单片机的 NRST 引脚。电路通过 10K 电阻上拉至 3.3V，并外接 100nF 电容接地，以实现硬件消抖。
* **低功耗唤醒**：配置了唤醒按键 K2，连接至单片机的 PA0/WAKEUP 引脚。配合外部下拉电阻设计，按下按键时输入高电平，可将芯片从待机等低功耗模式中唤醒。

### 5. 用户交互指示灯 (User LEDs)
* **引脚分配**：除了电源指示灯外，板子还配备了四个用户可编程控制的 LED 灯（LED2 至 LED5）。
* **驱动方式**：这四个 LED 分别由主控的 PA8、PB15、PB14 以及 PB13 引脚驱动。每个 LED 回路均串联了 1k 欧姆的限流电阻（如 R6 至 R9），当单片机对应引脚输出高电平时即可点亮指示灯。

### 6. 外设引脚扩展 (Expansion Headers)
* **全 IO 引出**：通过两侧的双排插针接口，将 MCU 几乎所有的 GPIO 端口全部引出，便于项目后期的外设扩展与杜邦线接线。
* **启动模式配置**：电路引出了 BOOT0 引脚，用户可以通过配置跳线帽电平，轻松选择从主闪存（Main Flash）还是系统存储器（System Memory / ISP 串口下载）启动。
